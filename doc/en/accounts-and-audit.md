# Accounts & audit

## Accounts (user management)

**Path:** Sidebar → **Accounts** — **Admin only**

Manage staff logins:

| Field | Description |
|-------|-------------|
| Name | Display name |
| Email | Username for sign-in |
| Role | Admin, Manager, Reception, Housekeeper, Restaurant, Kitchen |
| Status | Active / inactive |

After first-time setup, add accounts for each staff member instead of sharing the admin password.

## Messages

**Path:** Sidebar → **Messages** — **Admin only**

Communication log for guest and staff messages. Appears on dashboard as **Communication Log** / **Recent Activity**.

## Audit log

**Path:** Sidebar → **Audit Log** — **Admin only**

Compliance-oriented change history across the system. Complements per-module change logs.

## Per-module change logs

Most modules include a **Change Log** panel tracking edits, e.g.:

| Module | Log key |
|--------|---------|
| Rooms | Rooms change log |
| Bookings | Bookings change log |
| Guests | Guests change log |
| Housekeeping | Housekeeping change log |
| Invoices | Invoices change log |
| Inventory | Inventory / inventory audit |
| Restaurant | Restaurant change log |
| Order History | Order history change log |
| Accounts | Accounts change log |
| Dropdown Lists | Dropdowns change log |
| All Transactions | Transactions change log |

Empty logs show: *"No changes recorded yet"*

## Security practices

1. Disable inactive accounts instead of sharing credentials
2. Use least-privilege roles (Kitchen vs Admin)
3. Export audit data periodically for compliance
4. Change setup password after initial configuration if shared during install
