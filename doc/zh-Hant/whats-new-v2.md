# v2.3 / v2.4 中的新增功能

本指南總結了 HotelRestaurantMini-MartManagement 的 **stable v2.3** 和 **stable v2.4** 添加的主要功能。

## 21 種語言的完整介面

Web 應用程式 UI 支援 **21 種語言環境**：英語、西班牙語、法語、德語、日語、韓語、阿拉伯語、印地語、泰語、越南語、印尼語、土耳其語、俄語、義大利語、荷蘭語、波蘭語、希伯來語、寮語、葡萄牙語（巴西）、中文（簡體）和中文（繁體）。

### 在哪裡更改語言

|螢幕|如何|
|--------|-----|
| **登入/設定** |標題中的語言下拉清單（登入前）|
| **登入後** |頂部欄區域設定選擇器或選單中的**本地化** |
| **設定** |應用程式語言部分 |

首選項保存在瀏覽器儲存中 (`hotel_mgr_uiLocale`)。

### RTL（從右到左）

**阿拉伯語**和**希伯來語**為整個應用程式啟用 RTL 佈局。模態表單使用改進的對齊方式，因此標籤和輸入可以在 LTR 和 RTL 語言中正確讀取。

## 首次設定（已翻譯）

設定精靈已完全本地化：

- 企業/飯店名稱
- 系統標題文本
- 管理員使用者名稱、電子郵件和密碼字段
- 所有按鈕和驗證訊息

設定後，飯店名稱將儲存並顯示在配置的應用程式標題中。

## 翻譯後的情態形式

新增和編輯對話框已在地化為所有 21 種語言，包括：

- **維護** — 新票（房間、優先順序、問題、註釋）
- **發票** — 新增/編輯（客人、房間、日期、金額、付款狀態）
- **庫存** — 新增/編輯商品（名稱、條碼、類別、數量、POS 可用性）
- **選單項目** — 新增/編輯（名稱、圖示、價格、類別、圖片、庫存連結）
- **商店商品** — 新增/編輯（名稱、價格、類別、貨架圖示、條碼、庫存）- **使用者帳號** — 新增/編輯（姓名、電子郵件、密碼、角色）

圖像上傳標籤（「來自裝置」、「或圖像 URL」）遵循活動語言。

## 預訂 → 新客人

建立**預訂**時，如果客人尚未在目錄中：

1. 在預訂表上點選**+新客人**（或同等內容）。
2. 填寫**新客人**模式（姓名、護照、國籍、出生日期、付款方式、聯絡資訊、備註）。
3. 點擊 **新增客人並返回** — 您將返回預訂並選擇新客人。

國籍選擇器（搜尋清單）也已翻譯。
